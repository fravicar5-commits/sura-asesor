ℹ️ Notas sobre Comisiones

• PBS: comisión se paga una sola vez al momento de la afiliación, según rango de cotizantes y siniestralidad. No es recurrente.

• SPTI: 7% sobre la prima sin IVA cobrada — recurrente mientras el cliente pague cada período.

• SPD: 8% sobre la prima sin IVA cobrada — recurrente mientras el cliente pague cada período.

• Las comisiones de voluntarios se pagan cuando el cliente paga su prima — mes a mes si es mensual, o al cobro si es trimestral/semestral/anual.

• Los meses Enero, Mayo, Septiembre y Diciembre tienen factor X2 en el plan de reconocimiento.

• Bonos de consolación: $1.500.000 (Guatapé/Nuquí) · $2.500.000 (Convención Cancún).

📖 Cómo sincronizar en 5 pasos (sin ser programador)
1
Crear cuenta en GitHub
Ve a github.com → Sign up → Crea tu cuenta gratuita con tu correo.
2
Crear un repositorio nuevo
Clic en + New repository → Ponle nombre: sura-asesor-datos → Marca Private (privado) → Create repository.
3
Subir el archivo de datos
Desde tu repositorio → Add file → Upload files → Arrastra el sura_datos.json que descargaste → Commit changes.
4
En el otro dispositivo — Importar
Abre la app → Sincronización → Importar datos → selecciona el JSON descargado desde GitHub. Listo.
5
Rutina diaria recomendada
Al terminar el día en el celular: Exportar datos → Subir a GitHub. Al empezar en PC: Descargar JSON de GitHub → Importar. Así siempre tienes todo actualizado.
⚠️ Puntos importantes

• El repositorio Private protege tus datos — nadie más puede verlos.

• Si el repositorio es Public, el JSON es visible para todos — úsalo solo para pruebas.

• La sincronización es manual — tú decides cuándo exportar e importar.

• Si usas dos dispositivos al mismo tiempo, el último en importar sobreescribe los datos del otro.

• Para importar la app completa en un dispositivo nuevo, descarga el .html desde aquí y ábrelo en el navegador. Luego importa el JSON con tus datos.
